# ตารางเปรียบเทียบ REST และ GraphQL

## ผลการทดลอง

| หัวข้อเปรียบเทียบ   | REST (/api/v1/students/1/full)                                             | GraphQL                                                       |
| ------------------- | -------------------------------------------------------------------------- | ------------------------------------------------------------- |
| จำนวนคำขอ (Request) | 1 คำขอ                                                                     | 1 คำขอ                                                        |
| ข้อมูลที่ได้รับ     | ส่งข้อมูลทั้งหมดของนักศึกษา เช่น id, name, major, email, phone และ courses | ส่งเฉพาะฟิลด์ที่ระบุใน Query เช่น name และ courses.courseName |
| Over-fetching       | มี เพราะได้รับข้อมูลมากกว่าที่ต้องการ                                      | ไม่มี เพราะส่งเฉพาะข้อมูลที่ร้องขอ                            |
| ความยืดหยุ่น        | ต่ำ ผู้ใช้เลือกข้อมูลไม่ได้                                                | สูง ผู้ใช้เลือกเฉพาะฟิลด์ที่ต้องการได้                        |
| ปริมาณข้อมูลที่รับ  | มากกว่า เนื่องจากมีข้อมูลที่ไม่จำเป็น                                      | น้อยกว่า เพราะรับเฉพาะข้อมูลที่ต้องการ                        |

---

## สรุปผล

จากการทดลองพบว่า REST API ส่งข้อมูลทั้งหมดของนักศึกษา แม้ว่าจะต้องการเพียงบางฟิลด์ เช่น `name` และ `courses.courseName` จึงเกิดปัญหา **Over-fetching** เพราะได้รับข้อมูลมากเกินความจำเป็น

ส่วน GraphQL สามารถกำหนดฟิลด์ที่ต้องการได้ใน Query จึงได้รับเฉพาะข้อมูลที่ใช้งานจริง ทำให้ลดปริมาณข้อมูลที่ส่งผ่านเครือข่าย และสามารถดึงข้อมูลที่เกี่ยวข้องได้ภายในคำขอเดียว จึงมีความยืดหยุ่นมากกว่า REST API
