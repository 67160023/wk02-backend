# ตารางเปรียบเทียบ REST และ GraphQL

## ผลการทดลอง

| หัวข้อเปรียบเทียบ              | REST (/api/v1/students/1/full) | GraphQL                            |
| ------------------------------ | ------------------------------ | ---------------------------------- |
| จำนวนคำขอเพื่อดูข้อมูลของ1คน   | 1 คำขอ                         | 1 คำขอ                             |
| จำนวนคำขอเพื่อดูข้อมูลของทุกคน | มากกว่า1 คำขอ                  | 1 คำขอ                             |
| ข้อมูลที่ได้รับ                | ข้อมูลทั้งหมดของนักศึกษา       | เฉพาะฟิลด์ที่ระบุใน Query เท่านั้น |
| Over-fetching                  | มี เช่น email, phone           | ไม่มี                              |

---

## สรุปผล

REST API จะส่งข้อมูลทั้งหมดของนักศึกษา ถึงเราจะต้องการแค่บางฟิลด์ เช่น `name` และ `courses.courseName` จึงทำให้เกิดปัญหา **Over-fetching** เพราะได้รับข้อมูลมาเกินความจำเป็น

ส่วน GraphQL สามารถกำหนดฟิลด์ที่ต้องการได้ใน Query จึงได้รับเฉพาะข้อมูลที่ต้องการโดยไม่มีฟิลด์อื่นๆเพิ่มเติมจากที่ร้องขอไป และสามารถดึงข้อมูลที่เกี่ยวข้องได้ภายในคำขอเดียว
